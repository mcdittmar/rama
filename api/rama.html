rama.registry module¶

Data model concepts, i.e. types and roles, are represented by portable strings that are globally unique. Such identifiers can be used in different serialization formats to map data and metadata to the data model elements they represent.

VO-DML types are mapped to Python classes.

This module contains a TypeRegistry class that can be used to map VO-DML type identifiers to the corresponding Python class. This class is a singleton, which ensures that types are registered for the entire process life cycle.

The module also defines a VO class decorator that can be used to map Python classes to the VO-DML type they represent. When classes with the decorator are imported they are registered by the ID passed to the decorator.

Example:

>>> from rama.registry import VO, TypeRegistry
>>> @VO('foo:bar')
... class MyClass:
...   pass
...
>>> TypeRegistry.instance.get_by_id('foo:bar')
<class '....MyClass'>

Note that trying to instantiate the registry results in an exception:

>>> from rama.registry import TypeRegistry
>>> TypeRegistry()
Traceback (most recent call last):
  ...
TypeError: Singletons must be accessed through `instance`.
